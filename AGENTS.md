# Project Notes

This is a Vivado/Vitis Zynq-7000 SDR bring-up project for a single-channel
shortwave direct-sampling ADC path, currently focused on ADC input and AXI DMA
receive testing.

## Current Hardware Shape

- Vivado project: `HF_SDR.xpr`
- Device from routed timing report: `xc7z020-clg400-2`
- Exported hardware: `DataStreamTest.xsa`
- Main RTL:
  - `HF_SDR.srcs/sources_1/new/HF_SDR_top.v`
  - `HF_SDR.srcs/sources_1/new/adc_axis_source.v`
- Constraints: `HF_SDR.srcs/constrs_1/new/top.xdc`
- Block design generated wrapper: `HF_SDR.gen/sources_1/bd/design_1/hdl/design_1_wrapper.v`

The active PL data path is:

`ad_ch1[11:0] -> adc_axis_source -> AXIS FIFO -> AXI DMA S2MM -> PS DDR`

`ad_ch2` is present at the top-level but unused in the first receiver build.
DAC outputs are held at midscale and their clocks/write strobes are driven from
the PL clock so the analog outputs stay quiet during receive-only tests.

## ADC/AXIS Behavior

`adc_axis_source.v`:

- Assumes 12-bit ADC input.
- Defaults to offset-binary input conversion to signed two's complement.
- Left-aligns the 12-bit signed value into signed 16-bit samples.
- Uses `DECIM_BITS = 6`, so 65 MHz / 64 = 1.015625 MSPS.
- Emits 16-bit AXI-Stream samples with `tkeep = 2'b11`.
- Emits `tlast` every 512 samples.

Keep the PS DMA transfer length matched to the TLAST cadence. With the current
512 samples x 2 bytes format, one packet is 1024 bytes.

## Vitis Test App

Main bare-metal test app:

- `vitis/StreamTest/src/main.c`

The app:

- Uses AXI DMA simple mode, S2MM only.
- DMA base is `XPAR_XAXIDMA_0_BASEADDR` / hardware address `0x40400000`.
- Captures 32 packets.
- Receives 1024 bytes per packet into a 64-byte aligned buffer.
- Flushes before DMA, invalidates after DMA.
- Prints min/max/avg/changed count and first 16 samples.
- Polls S2MM status and resets DMA on error.

## Practical Debug Notes

- If DMA times out, first check whether TLAST still arrives every 1024 bytes and
  whether the S2MM transfer length matches that exact packet size.
- If DMA returns S2MM error bits, inspect `S2MM_DMASR` before resetting; the app
  already prints the raw status.
- If samples are constant, distinguish three cases:
  - ADC input is actually static or biased.
  - ADC bit order / offset-binary setting is wrong.
  - AXIS handshaking is stalled and repeatedly exposing old samples.
- If sample values look inverted around midscale, re-check `ADC_OFFSET_BINARY`.
- Before higher-confidence ADC testing, add proper ADC input delay constraints
  relative to the ADC sampling/output clock relationship.
